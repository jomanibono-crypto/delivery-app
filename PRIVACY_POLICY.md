## 2. Data Collected

### 2.1 Location Data (GPS)
- **Continuous real-time GPS position** while the app is in use or running in the background.
- **Purpose:** Share the user's location with other members of the same group so they can see each other on a shared map.
- **Retention:** Location data is written to Firebase Realtime Database and deleted immediately when the user leaves the group or disconnects (via `onDisconnect().remove()`). History points (route traces) are persisted until cleaned up by expiration logic (12-hour default) or manual group deletion.
- **Background collection:** The app uses a foreground service with a persistent low-importance notification to continue collecting location data when the app is closed. Users can stop background collection at any time by leaving the group or disabling location permissions.

### 2.2 Account Data
- **Anonymous Firebase Authentication:** The app uses Firebase Anonymous Authentication. No email, password, phone number, or third-party identity is required. A unique anonymous UID is generated by Firebase.
- **Display name:** The user provides a self-chosen display name (stored in Firebase Realtime Database and local SharedPreferences).
- **Avatar icon:** The user selects an emoji avatar (stored in Firebase Realtime Database and local SharedPreferences).

### 2.3 Messages
- **Chat messages:** Text messages sent via the in-app group chat are stored in Firebase Realtime Database and visible to all group members. Messages older than 1 hour are automatically deleted.
- **Alert markers:** Location-based alert markers (police, speed trap, control points, notes, bad customer reports) placed by the user on the shared map are stored in Firebase Realtime Database. Markers older than 12 hours are automatically deleted.

### 2.4 Phone Numbers (Blacklist)
- **Phone numbers added to the blacklist:** If the user adds a phone number to the shared group blacklist, the number and a reason are stored in Firebase Realtime Database and visible to all authenticated group members.

### 2.5 Crash Data
- **Firebase Crashlytics:** The app uses Firebase Crashlytics to collect crash reports, non-fatal errors, and device information (device model, OS version, app version) for debugging and stability purposes. This data is not used for advertising or profiling.

### 2.6 Device Information
- **App version, device model, OS version:** Collected by Firebase Crashlytics and Firebase Realtime Database for operational purposes.

## 3. Legal Basis (GDPR)

Under the General Data Protection Regulation (GDPR), the legal basis for processing location data is:
- **Consent (Article 6(1)(a)):** The user explicitly grants location permission via the Android permission system.
- **Legitimate interest (Article 6(1)(f)):** Processing is necessary for the core functionality of the app (real-time location sharing within a user-created group).

## 4. Data Sharing

Data is shared with:
- **Firebase (Google):** Realtime Database, Crashlytics, App Check, Authentication. Google's privacy policy applies: https://policies.google.com/privacy
- **Mapbox:** Map tiles are loaded from Mapbox when using the primary tile provider. Mapbox's privacy policy: https://www.mapbox.com/privacy
- **Other group members:** Location data, name, avatar, chat messages, and alert markers are visible to other authenticated members of the same group.

## 5. Data Retention

| Data Type | Retention Period |
|-----------|-----------------|
| Real-time GPS location | Deleted immediately on disconnect (onDispose/onDisconnect) |
| Route history points | 12 hours max |
| Chat messages | 1 hour max |
| Alert markers | 12 hours max |
| Blacklist entries | Until manually deleted by any group member |
| App version metadata | Indefinitely (publish history) |

## 6. User Rights (GDPR)

Users have the right to:
- **Access:** Request a copy of their data.
- **Rectification:** Correct inaccurate data (e.g., change display name in settings).
- **Erasure:** Delete their data by leaving the group (removes location data immediately) or contacting the developer to request full Firebase deletion.
- **Restrict processing:** Deny location permissions at the OS level.
- **Data portability:** Contact the developer to request exported data.
- **Object:** Stop using the app at any time.

To exercise these rights, contact: yassinx2001@gmail.com

## 7. Data Security

- Firebase App Check (Play Integrity) verifies that requests come from the genuine app.
- All Firebase communication is encrypted via HTTPS/TLS.
- Firebase Realtime Database security rules restrict access to authenticated group members.
- Anonymous authentication means no passwords are stored.
- APK downloads are verified via SHA-256 hash.
